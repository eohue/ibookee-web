# 📦 Render 하나로 끝내기 (가장 쉬운 통합 솔루션)

"여러 서비스를 쓰니까 복잡해요. 그냥 하나로 다 하고 싶어요."
라는 고민에 대한 가장 명쾌한 해답은 **현재 사용 중인 Render에서 모든 기능을 켜는 것**입니다.

AWS나 다른 곳으로 갈 필요 없이, Render 안에서 **데이터베이스**와 **이미지 저장소(디스크)**를 모두 해결할 수 있습니다.

---

## 1. 어떻게 바뀌나요?

| 기능 | 현재 | **Render 통합 후** | 장점 |
| :--- | :--- | :--- | :--- |
| **웹사이트** | Render Web Service | **Render Web Service** | 그대로 유지 |
| **데이터베이스** | Neon | **Render PostgreSQL** | 같은 사이트에서 관리, 속도 향상 |
| **이미지 저장** | (없음/AWS S3 예정) | **Render Persistent Disk** | **코드 수정 없음**, S3 가입 불필요 |

👉 **결과:** Neon도, AWS도 필요 없습니다. 오직 **Render** 사이트 하나만 로그인하면 됩니다.

---

## 2. 'Render 디스크'란 무엇인가요? (이미지 문제 해결사)
지금 겪고 계신 "이미지가 자꾸 사라지는 문제"를 해결하기 위해 굳이 복잡한 AWS S3를 쓸 필요가 없습니다.
Render에는 **[Persistent Disk (영구 디스크)]**라는 기능이 있습니다.

*   **원리:** 서버에 "지워지지 않는 외장 하드"를 하나 꽂아주는 것입니다.
*   **설정:** "이 폴더(`attached_assets`)는 지우지 말고 저장해줘"라고 설정만 하면 끝입니다.
*   **장점:** 코드를 단 한 줄도 고칠 필요가 없습니다.

---

## 3. 통합 시 비용 체크 (중요!)
편리함에는 비용이 따릅니다. Render의 무료 플랜(Free)에서는 이 고급 기능들을 쓸 수 없습니다.

*   **기존**: Render(무료) + Neon(무료) = **0원** (하지만 이미지 사라짐)
*   **통합 후 예상 비용**:
    1.  **Web Service (Starter)**: 약 $7/월 (디스크를 쓰려면 유료 플랜 필수)
    2.  **Persistent Disk**: 약 $0.25/월 (1GB 기준)
    3.  **PostgreSQL (선택)**: 약 $7/월 (Neon을 계속 써도 되지만 통합하고 싶다면)
    *   **총합**: **약 월 1~2만원 수준**

> 💡 **결론**: "월 1만원 정도는 낼 수 있으니, 제발 복잡한 거 없이 쉽고 편하게 가고 싶다!" 하시면 이 방법이 **최고의 선택**입니다.

---

## 4. 실행 가이드 (이렇게 하세요)

### 1단계: Render 유료 플랜 업그레이드
1.  Render 대시보드에서 사용 중인 웹 서비스(ibookee)를 클릭합니다.
2.  **Settings** -> **Upgrade to Starter**를 눌러 유료 플랜($7)으로 변경합니다. (결제 카드 등록 필요)

### 2단계: 디스크 연결 (이미지 문제 해결!)
1.  같은 메뉴의 **Disks** 탭을 클릭합니다.
2.  **Add Disk**를 누릅니다.
3.  설정값 입력:
    *   **Name**: `ibookee-assets` 등 자유롭게
    *   **Mount Path**: `/opt/render/project/src/attached_assets` (매우 중요! 정확히 입력)
    *   **Size**: `1 GB` (충분합니다)
4.  저장하면 서버가 재부팅됩니다. **이제 이미지는 영원히 사라지지 않습니다.**

### 3단계 (선택): 데이터베이스 가져오기
Neon도 없애고 싶으시다면 Render에서 PostgreSQL을 생성하고 데이터를 옮기면 됩니다. 하지만 Neon이 성능이 좋고 무료 범위가 넓어서, **DB는 Neon에 그냥 두셔도 무방합니다.** (이미지 문제와는 관련이 없으니까요!)
